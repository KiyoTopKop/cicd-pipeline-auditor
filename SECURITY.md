# Security Policy

## 🛡️ Reporting a Vulnerability

We take the security of PipeLint seriously. If you believe you've found a security vulnerability in our tool, please do **not** report it via a public issue. 

Instead, please send an email to: **kiyotopkopy@gmail.com**

We will acknowledge your report within 48 hours and provide a timeline for fixes.

## 🤝 Responsible Disclosure

We ask that you follow these principles:

- **Do not disclose** the vulnerability publicly until we have had a reasonable amount of time to address it.
- **Provide a proof of concept** (PoC) if possible to help us reproduce the issue.
- **Do not perform** any destructive actions or access unauthorized data as part of your research.

## 📦 Scope

This policy applies specifically to the `pipelint` engine and its distribution (CLI, Web UI, Action). 

Note: PipeLint is a static analysis tool designed to detect security issues in *other* projects. Findings in specific third-party workflows are not considered vulnerabilities in PipeLint itself unless they expose a bypass in our engine.

---

Thank you for helping keep the PipeLint community safe!
